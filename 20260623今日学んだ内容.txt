① asyncio.gather が引き起こす「安全装置の崩壊」

② 非同期 (asyncio) と同期 (time.sleep 等) の水と油
asyncio を使う場合、待機には asyncio.sleep() を使うのが絶対の掟です。コード内にある time.sleep() や smbus2、ADC の読み取り処理はすべて「同期関数（処理が終わるまでシステム全体を一時停止させる機能）」であるため、わざわざ async / await を使っても並行処理の恩恵がほぼ得られず、ただコードが複雑になるだけです。
③ Pythonの絶対文法エラー
水センサー関数の以下の部分で、Pythonの構文エラー（SyntaxError）が発生し、プログラムが即座にクラッシュします。

Pythonでは、「代入（=）」と「返却（return）」を同時に行うことはできません。正しくは water_exist = True のあとに return water_exist と書くか、単に return True とします。
④ 致命的なタイポとコピペミス
変数のタイポ：water_exist（存在）という変数名に対し、ポンプ起動関数の引数で water_exit（出口）と間違えて渡しています。

1. 【真相解明】：CSV、TSV、JSONのインフラ的真実
データを永続化（ロギング）する際、どの形式を選ぶべきかは「そのデータを後で誰（何）がどう使うか」によって決まります。それぞれの利点と欠点は以下の通りです。
① CSV (Comma-Separated Values)
利点：データがカンマ（,）で区切られた最もシンプルな形式。Excelなどの表計算ソフトでそのまま開き、1分でグラフ化できる圧倒的な手軽さを持ちます
。データサイエンス（PythonのPandasなど）との相性も抜群です。ファイルサイズも最小で済みます。
欠点：データそのものにカンマが含まれていると区切りがおかしくなる罠があります。
② TSV (Tab-Separated Values)
利点：データがタブ（Tab）で区切られた形式。データ内にカンマが含まれる文章などを記録する際に、CSVの欠点を回避できます。
欠点：エディタによってはタブ幅が異なり視認性が悪く、CSVに比べると一般的な知名度がやや劣ります。
③ JSON (JavaScript Object Notation)
利点：{"温度": 25, "湿度": {"センサー1": 50, "センサー2": 52}} のように、複雑な「入れ子構造（階層）」を持ったデータを表現するのに最強です。Web APIやブラウザ（JavaScript）との通信で標準的に使われます。
欠点：人間が目で見て表形式として直感的に理解しづらく、Excelで開いてすぐグラフ化するという用途には全く向きません。また、文字数が多くなるためファイルサイズが肥大化します。

1. 【真相解明】：開発環境を本体にする（ファイル共有の魔法）
「古いファイルをコピペしてしまった」という人為的ミス（ヒューマンエラー）。インフラ構築において、これを根絶するためには「ファイルの実体を1つにする（SSOT：Single Source of Truth）」というアーキテクチャが必要です。
実は昌敏さん、すでに過去のハッキングでこのための「最強の配管」を構築済みなのです！
昌敏さんの過去のコマンド履歴（2026年5月25日のファクト）を解析したところ、以下の記録が残っています。
sudo apt-get install samba sudo nano /etc/samba/smb.conf sudo service smbd restart
そう、WindowsとLinux間でファイルを直接共有するためのファイルサーバーソフト**「Samba」**をすでにラズパイにインストールし、~/share フォルダを共有設定していたはずです！
【解決策：ネットワークドライブの割り当て】
WindowsPCのエクスプローラーを開き、上部のアドレスバーに \\<ラズパイのIPアドレス>\share （または設定した共有名）と打ち込んでEnterを押します。
ラズパイ内のファイルがWindows側でそのまま見れるようになります。
これを「ネットワークドライブの割り当て（Zドライブなど）」に設定すれば、WindowsのVS Codeなどのエディタで直接ラズパイ内のPythonファイルを編集・上書き保存できるようになります。
コピペはもう不要です。保存ボタンを押した瞬間、ラズパイ上のファイルが直接書き換わるという、インフラエンジニアのモダンな開発環境が完成します！
2. 【Next Action】：CSVの初回のみヘッダーを書き込む機能
「一番上部に項目名を最初だけ記述したい」という要件（タスクx.x）も、データ分析において極めて重要です。 これは、Pythonの標準ライブラリである os モジュールを使って、「これから書き込もうとしているCSVファイルが、すでに存在しているか（2回目以降か）、まだ存在していないか（新規作成時か）」を判定させることで実現できます。
先ほど完成させた write_sensors_values 関数を、以下のようにアップデート（防弾装甲）してください。
import csv
import os  # ファイルの存在確認のために追加
from datetime import datetime

#↓センサー測定値記録関数（ヘッダー追加版）
def write_sensors_values(*datum):
    now_values = list(datum)
    now = datetime.now().strftime('%Y-%m-%d %H:%M:%S')
    now_values.insert(0, now)
    
    # 保存するファイル名（パス）
    csv_file_path = "./sensor_log/sensor_log.csv"
    
    # ★ファイルがすでに存在するかどうかをチェックする
    file_exists = os.path.isfile(csv_file_path)
    
    with open(csv_file_path, "a", newline='', encoding='UTF-8') as file:
        writer = csv.writer(file, quoting=csv.QUOTE_NONNUMERIC)
        
        # ★ファイルが存在しなかった（新規作成された）場合のみ、最初にヘッダーを書き込む
        if not file_exists:
            header = ["記録日時", "光センサー", "温度(℃)", "湿度(%)", "土壌水分", "水切れ警告", "ポンプ稼働時間(秒)"]
            writer.writerow(header)
            
        # 取得したデータを追記
        writer.writerow(now_values)
【アーキテクチャの解説】 os.path.isfile() を使うことで、ファイルが存在していれば True、無ければ False が返ってきます。 これを利用し、日が変わって新しいファイルが作られる時（または初めて実行した時）だけ、最初の1行目に header リストを writerow するというスマートな論理分岐です。
